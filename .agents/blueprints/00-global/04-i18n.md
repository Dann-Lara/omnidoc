## 🏗️ Sistema Multi-nivel de Idiomas

### Flujo de Resolución de Idioma

```
[PRECEDENCIA - Mayor a Menor]

1. localStorage.omnidoc-lang      → Override del usuario (navbar)
2. Organization.settings.lang    → Idioma default del tenant
3. SAAS global lang            → Idioma default del sistema
4. localStorage (default SAAS) → Fallback si no hay tenant
```

### Dónde se guarda cada nivel

| Nivel | Campo | Ubicación | Quien puede cambiar |
|-------|------|----------|-------------------|
| SAAS Global | `localStorage.omnidoc-lang` | Landing | SUPERADMIN (`/admin/settings`) |
| Org Default | `Organization.settings.lang` | DB | OWNER (`/[slug]/settings`) |
| User Override | `localStorage.omnidoc-lang` | Navbar | Cualquier usuario |

## 🔧 Cómo usar el sistema

### En Frontend

```tsx
import { useI18n } from '@/lib/i18n'

export function MyComponent() {
  const { lang, t } = useI18n()
  
  return <h1>{t('admin.nav.dashboard')}</h1>
}
```

### En Backend (API)

```typescript
import { t } from '@/i18n/translations'

throw new NotFoundException(t('errors.team.memberNotFound', lang))
```

### En Mails

```typescript
await this.mailService.sendInvitationEmail({
  to: email,
  lang: orgSettings.lang, // ← usar lang de la Org
  // ...
})
```

## ⚠️ Reglas Importantes

1. **No hardcodear textos** - Siempre usar `t('path')`
2. **Usar paths descriptivos** - `admin.nav.dashboard` no `and`
3. **Mantener ambos idiomas** - Cada clave debe tener EN y ES
4. **No usar isSpanish directo** - Usar siempre `useI18n()`
5. **Lang dinámico** - Pasar lang de la Org al crear recursos
